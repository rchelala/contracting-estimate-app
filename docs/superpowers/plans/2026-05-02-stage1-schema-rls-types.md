# Stage 1 — Schema, RLS & TypeScript Types Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the full Supabase Postgres schema, RLS policies, DB functions, and generated TypeScript types for EstimateFlow — no UI, no API routes, just the data layer.

**Architecture:** Numbered SQL migration files under `supabase/migrations/` are applied via the Supabase CLI. Each migration is self-contained: it creates its table(s), indexes, and RLS policies together. Two Postgres helper functions (`is_org_member`, `is_org_owner`) are created first so every subsequent RLS policy can reference them. Business-logic functions (`recalculate_estimate_totals`, `next_estimate_number`, `next_invoice_number`) live in a dedicated migration. TypeScript types are generated from the live local DB after all migrations run.

**Tech Stack:** Supabase CLI, PostgreSQL 15, `@supabase/supabase-js` v2, Vite + React 18 + TypeScript strict, Vitest (unit tests for `money.ts`)

## Task 4: Migration — RLS Helper Functions

**Files:**
- Create: `supabase/migrations/20260502000002_rls_helpers.sql`

These two functions are called by every RLS policy in later migrations. They must exist first.

- [ ] **Step 1: Write the migration**

Create `supabase/migrations/20260502000002_rls_helpers.sql`:

```sql
-- Returns true if the currently authenticated user belongs to the given org.
-- SECURITY DEFINER so the function can query organization_members without
-- the caller needing direct SELECT on that table.
CREATE OR REPLACE FUNCTION is_org_member(p_org_id uuid)
RETURNS boolean
LANGUAGE sql
SECURITY DEFINER
STABLE
AS $$
  SELECT EXISTS (
    SELECT 1
    FROM organization_members
    WHERE organization_id = p_org_id
      AND user_id = auth.uid()
  );
$$;

-- Returns true if the currently authenticated user is the owner of the given org.
CREATE OR REPLACE FUNCTION is_org_owner(p_org_id uuid)
RETURNS boolean
LANGUAGE sql
SECURITY DEFINER
STABLE
AS $$
  SELECT EXISTS (
    SELECT 1
    FROM organization_members
    WHERE organization_id = p_org_id
      AND user_id = auth.uid()
      AND role = 'owner'
  );
$$;
```

```sql
CREATE TABLE organizations (
  id                          uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
  name                        text        NOT NULL,
  logo_storage_path           text,
  default_tax_zip             text,
  platform_fee_pct            numeric(5,2) NOT NULL DEFAULT 0,
  stripe_account_id           text,

                                CHECK (plan IN ('free', 'pro')),
  plan_period_start           timestamptz,
  created_at                  timestamptz NOT NULL DEFAULT now(),
  updated_at                  timestamptz NOT NULL DEFAULT now()
);

CREATE TABLE organization_members (
  id               uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
  organization_id  uuid        NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  user_id          uuid        NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  role             member_role NOT NULL DEFAULT 'member',
  created_at       timestamptz NOT NULL DEFAULT now(),
  UNIQUE (organization_id, user_id)
);

CREATE INDEX organization_members_user_id_idx ON organization_members (user_id);
CREATE INDEX organization_members_org_id_idx  ON organization_members (organization_id);

-- RLS
ALTER TABLE organizations         ENABLE ROW LEVEL SECURITY;
ALTER TABLE organization_members  ENABLE ROW LEVEL SECURITY;

-- organizations: any member can read their org
CREATE POLICY "members can read own org"
  ON organizations FOR SELECT
  USING (is_org_member(id));

-- organizations: only owner can update
CREATE POLICY "owner can update org"
  ON organizations FOR UPDATE
  USING (is_org_owner(id));

-- organization_members: members can read the member list of their org
CREATE POLICY "members can read org members"
  ON organization_members FOR SELECT
  USING (is_org_member(organization_id));

-- organization_members: only owner can insert/delete members
CREATE POLICY "owner can manage members"
  ON organization_members FOR INSERT
  WITH CHECK (is_org_owner(organization_id));

CREATE POLICY "owner can delete members"
  ON organization_members FOR DELETE
  USING (is_org_owner(organization_id));
```

## Task 7: Migration — Estimates

**Files:**
- Create: `supabase/migrations/20260502000005_estimates.sql`

This is the largest migration. It includes the per-org sequence table used to generate `EST-0001` style numbers, and two RLS policies: one for org members and one that allows unauthenticated access via the public token (for the client-facing estimate view).

- [ ] **Step 1: Write the migration**

Create `supabase/migrations/20260502000005_estimates.sql`:

```sql
-- Per-org counter for estimate numbers. Each org gets one row.
-- The next_estimate_number() function (Task 15) does an atomic increment here.
CREATE TABLE estimate_sequences (
  organization_id  uuid     PRIMARY KEY REFERENCES organizations(id) ON DELETE CASCADE,
  last_number      integer  NOT NULL DEFAULT 0
);

CREATE TABLE estimates (
  id                    uuid             PRIMARY KEY DEFAULT gen_random_uuid(),
  organization_id       uuid             NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  client_id             uuid             REFERENCES clients(id) ON DELETE SET NULL,
  estimate_number       text             NOT NULL,  -- e.g. EST-0001
  status                estimate_status  NOT NULL DEFAULT 'draft',
  title                 text,
  notes                 text,
  -- Tax (snapshotted when sent; never recalculated on sent estimates)
  tax_zip               text,
  tax_rate_pct          numeric(6,4),
  -- Server-computed totals (cents). Client values are advisory.
  subtotal_cents        integer          NOT NULL DEFAULT 0,
  tax_cents             integer          NOT NULL DEFAULT 0,
  total_cents           integer          NOT NULL DEFAULT 0,
  -- Public client-facing link token (32 URL-safe chars)
  public_token          text             NOT NULL UNIQUE DEFAULT encode(gen_random_bytes(24), 'base64'),
  -- Timestamps
  issued_at             timestamptz,
  expires_at            timestamptz,
  sent_at               timestamptz,
  first_sent_at         timestamptz,   -- set once, on first send
  -- Approval fields
  approved_at           timestamptz,
  approved_by_name      text,
  approved_client_ip    text,
  approved_user_agent   text,
  created_at            timestamptz      NOT NULL DEFAULT now(),
  updated_at            timestamptz      NOT NULL DEFAULT now(),
  UNIQUE (organization_id, estimate_number)
);

CREATE INDEX estimates_org_id_idx    ON estimates (organization_id);
CREATE INDEX estimates_client_id_idx ON estimates (client_id);
CREATE INDEX estimates_status_idx    ON estimates (status);
CREATE INDEX estimates_public_token_idx ON estimates (public_token);

ALTER TABLE estimates          ENABLE ROW LEVEL SECURITY;
ALTER TABLE estimate_sequences ENABLE ROW LEVEL SECURITY;

-- Org members: full CRUD on their estimates
CREATE POLICY "members can read own estimates"
  ON estimates FOR SELECT
  USING (is_org_member(organization_id));

CREATE POLICY "members can insert estimates"
  ON estimates FOR INSERT
  WITH CHECK (is_org_member(organization_id));

CREATE POLICY "members can update estimates"
  ON estimates FOR UPDATE
  USING (is_org_member(organization_id));

CREATE POLICY "members can delete estimates"
  ON estimates FOR DELETE
  USING (is_org_member(organization_id));

-- Public (unauthenticated) access via public_token — read only
-- Used by the client-facing /e/:token view
CREATE POLICY "public can read estimate by token"
  ON estimates FOR SELECT
  USING (public_token IS NOT NULL);

-- estimate_sequences is internal; only the service role / SECURITY DEFINER functions touch it
CREATE POLICY "service role manages sequences"
  ON estimate_sequences FOR ALL
  USING (false)
  WITH CHECK (false);
```

```sql
-- Recomputes subtotal, tax, and total for an estimate from its line items.
-- Called after every save. SECURITY DEFINER so it can update estimates
-- regardless of the calling user's RLS context.
CREATE OR REPLACE FUNCTION recalculate_estimate_totals(p_estimate_id uuid)
RETURNS void
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
  v_subtotal_cents  bigint;
  v_tax_rate_pct    numeric(6,4);
  v_tax_cents       bigint;
BEGIN
  -- Sum required (non-optional) line items:
  -- line total = ROUND(quantity * unit_price_cents * (1 + markup_pct / 100))
  SELECT COALESCE(SUM(
    ROUND(quantity * unit_price_cents * (1.0 + markup_pct / 100.0))::bigint
  ), 0)
  INTO v_subtotal_cents
  FROM estimate_line_items
  WHERE estimate_id = p_estimate_id
    AND optional = false;

  -- Retrieve the snapshotted tax rate
  SELECT tax_rate_pct
  INTO v_tax_rate_pct
  FROM estimates
  WHERE id = p_estimate_id;

  -- Tax applies only to taxable, non-optional line items
  SELECT COALESCE(SUM(
    ROUND(
      quantity * unit_price_cents * (1.0 + markup_pct / 100.0)
      * COALESCE(v_tax_rate_pct, 0) / 100.0
    )::bigint
  ), 0)
  INTO v_tax_cents
  FROM estimate_line_items
  WHERE estimate_id = p_estimate_id
    AND optional = false
    AND taxable = true;

  UPDATE estimates
  SET
    subtotal_cents = v_subtotal_cents,
    tax_cents      = v_tax_cents,
    total_cents    = v_subtotal_cents + v_tax_cents,
    updated_at     = now()
  WHERE id = p_estimate_id;
END;
$$;

-- Atomically increments the per-org estimate counter and returns the next number
-- as a formatted string (EST-0001, EST-0002, ...).
-- Uses INSERT ... ON CONFLICT to create the sequence row on first call.
CREATE OR REPLACE FUNCTION next_estimate_number(p_org_id uuid)
RETURNS text
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
  v_next integer;
BEGIN
  INSERT INTO estimate_sequences (organization_id, last_number)
  VALUES (p_org_id, 1)
  ON CONFLICT (organization_id)
  DO UPDATE SET last_number = estimate_sequences.last_number + 1
  RETURNING last_number INTO v_next;

  RETURN 'EST-' || LPAD(v_next::text, 4, '0');
END;
$$;

-- Same for invoices (INV-0001, INV-0002, ...).
CREATE OR REPLACE FUNCTION next_invoice_number(p_org_id uuid)
RETURNS text
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
  v_next integer;
BEGIN
  INSERT INTO invoice_sequences (organization_id, last_number)
  VALUES (p_org_id, 1)
  ON CONFLICT (organization_id)
  DO UPDATE SET last_number = invoice_sequences.last_number + 1
  RETURNING last_number INTO v_next;

  RETURN 'INV-' || LPAD(v_next::text, 4, '0');
END;
$$;
```

```typescript
export function applyMarkup(unitPriceCents: number, markupPct: number): number {

export function lineItemTotal(

export function formatCents(cents: number, currencyCode = 'USD'): string {

    style: 'currency',
    currency: currencyCode,
```
