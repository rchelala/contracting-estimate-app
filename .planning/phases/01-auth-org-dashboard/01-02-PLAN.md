# Plan 02: Auth Shell

## Objective

Build the auth lifecycle for EstimateFlow: a `useAuth` hook that mirrors Supabase session state, a `RequireAuth` route guard, the `/auth` page using `@supabase/auth-ui-react` with `ThemeSupa` (D-01), an `/auth/callback` handler that token-exchanges and routes new vs. returning users (D-04, D-06), and the React Router v6 route tree wiring everything together. The dashboard and onboarding pages are stubbed in this plan with placeholder content; their bodies are written in Plans 03 and 04.

This plan delivers AUTH-01 through AUTH-05, ORG-04 (RLS-only because session is in place), and DASH-01 (the protected route + redirect entry).

## Threat Model

<threat_model>

### Trust Boundaries
| Boundary | Description |
|----------|-------------|
| Browser localStorage ↔ React state | Supabase persists tokens in localStorage; XSS would extract them |
| /auth/callback URL params ↔ Supabase token exchange | Untrusted query string consumed by `detectSessionInUrl` |

### STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | Email verification link redirect | mitigate | `redirectTo` is bound to `${window.location.origin}/auth/callback`; allowed list is enforced in `supabase/config.toml` (Plan 01) |
| T-02-02 | Tampering | URL token in /auth/callback | mitigate | Token exchange handled by supabase-js (`detectSessionInUrl: true`); we never parse it manually |
| T-02-03 | Information Disclosure | Session token in localStorage | accept | Standard Supabase behavior; mitigated at app level by enforcing CSP in a future hardening pass and never rendering tokens in DOM |
| T-02-04 | Repudiation | Sign-out without server invalidation | mitigate | Use `supabase.auth.signOut()` (server-side revoke) — not client-only state clear |
| T-02-05 | Elevation of Privilege | RequireAuth bypass via direct route | mitigate | Guard is rendered by router; nested `<Outlet />` only renders when session is present. RLS is the actual security boundary; the guard is a UX layer |
| T-02-06 | Information Disclosure | Membership check in callback leaks org existence | accept | Query is scoped to current user via RLS; only reveals membership for auth.uid() |
| T-02-07 | DoS | Brute force sign-in | transfer | Supabase Auth rate-limits sign-in attempts server-side |
| T-02-08 | Tampering | XSS injection of org name into Auth UI | mitigate | Auth UI does not render arbitrary user-supplied org names; copy is static; React escapes by default |

</threat_model>

    ```ts
    import { useEffect, useState, useCallback } from 'react'
    import type { Session } from '@supabase/supabase-js'
    import { supabase } from '../lib/supabase'

    export interface UseAuthResult {
      session: Session | null
      loading: boolean
      signOut: () => Promise<void>
    }

    export function useAuth(): UseAuthResult {
      const [session, setSession] = useState<Session | null>(null)
      const [loading, setLoading] = useState<boolean>(true)

      useEffect(() => {
        let active = true

        supabase.auth.getSession().then(({ data }) => {
          if (!active) return
          setSession(data.session)
          setLoading(false)
        })

        const { data: { subscription } } = supabase.auth.onAuthStateChange((_event, next) => {
          setSession(next)
        })

        return () => {
          active = false
          subscription.unsubscribe()
        }
      }, [])

      const signOut = useCallback(async () => {
        await supabase.auth.signOut()
      }, [])

      return { session, loading, signOut }
    }
    ```

    3. Create `src/pages/AuthCallback.tsx` per UI-SPEC `/auth/callback`:
       ```tsx
       import { useEffect } from 'react'
       import { useNavigate } from 'react-router-dom'
       import { supabase } from '../lib/supabase'

       export default function AuthCallback() {
         const navigate = useNavigate()

         useEffect(() => {
           let cancelled = false
           async function run() {
             const { data: { session } } = await supabase.auth.getSession()
             if (cancelled) return
             if (!session) {
               navigate('/auth', { replace: true })
               return
             }
             const { data: membership, error } = await supabase
               .from('organization_members')
               .select('id')
               .eq('user_id', session.user.id)
               .limit(1)
               .maybeSingle()
             if (cancelled) return
             if (error) {
               // Fail closed: send to onboarding so user can set up org
               navigate('/onboarding', { replace: true })
               return
             }
             navigate(membership ? '/dashboard' : '/onboarding', { replace: true })
           }
           run()
           return () => { cancelled = true }
         }, [navigate])

         return (
           <div className="min-h-screen flex items-center justify-center bg-slate-50">
             <p className="text-sm text-slate-500">Verifying your email...</p>
           </div>
         )
       }
       ```
