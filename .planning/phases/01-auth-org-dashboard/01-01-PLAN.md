# Plan 01: Foundation

## Objective

Bring the Stage 1 data layer to master, install every frontend dependency Phase 1 (and forward Phase 2) needs, configure Tailwind CSS v4 with the Vite plugin, add the missing `create_organization` SECURITY DEFINER RPC migration (closing the documented INSERT-policy gap on `organizations`), update local Supabase config to match Vite's port 5173, create `vercel.json` with the SPA rewrite, and push the schema so downstream plans can build against a live database.

## Threat Model

<threat_model>

### Trust Boundaries
| Boundary | Description |
|----------|-------------|
| Authenticated browser → Postgres | RLS enforces tenant isolation; SECURITY DEFINER RPC bypasses RLS deliberately for first-org bootstrap |
| Anonymous browser → Vercel SPA | All routes serve index.html; auth is enforced client-side and again by RLS |

### STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of Privilege | `create_organization` RPC | mitigate | Function is `SECURITY DEFINER` but checks `auth.uid() IS NOT NULL` and uses the resolved `auth.uid()` (NOT a parameter) as the owner of the new org/member rows; cannot be used to elevate another user |
| T-01-02 | Tampering | `create_organization(p_name)` input | mitigate | Trim `p_name`, enforce length 1..120 via CHECK inside function and `RAISE EXCEPTION` on violation |
| T-01-03 | Information Disclosure | `vercel.json` SPA rewrite | accept | Static SPA serves no secrets; rewrite only affects unmatched routes — `/api/*` not rewritten |
| T-01-04 | Spoofing | Supabase site_url misconfiguration | mitigate | Add `http://localhost:5173/auth/callback` to `additional_redirect_urls` so Supabase will only honor that exact callback URL |
| T-01-05 | Denial of Service | RPC abuse (mass org creation) | accept | Free tier rate limits and AUTH checks are enough for MVP; revisit if abuse observed |
| T-01-06 | Repudiation | Unlogged org creation | accept | `organizations.created_at` and `organization_members.created_at` provide an audit trail; full audit log is post-MVP |

</threat_model>

    Create file `supabase/migrations/20260502000012_create_organization_rpc.sql` with exactly this content:

    ```sql
    -- create_organization: bootstrap RPC that lets an authenticated user create
    -- their first organization plus their owner membership in a single atomic step.
    --
    -- Why SECURITY DEFINER: the `organizations` table has SELECT/UPDATE policies but
    -- intentionally no INSERT policy — direct inserts from the authenticated role
    -- would 403. This function runs with definer privileges to bypass that policy
    -- for the bootstrap case while still binding both rows to auth.uid().
    --
    -- Hardening:
    --   * Requires a valid auth.uid() (rejects anonymous calls).
    --   * Trims and length-checks the company name (1..120 chars).
    --   * Owner is always auth.uid() — caller cannot spoof another user.

    CREATE OR REPLACE FUNCTION public.create_organization(p_name text)
    RETURNS uuid
    LANGUAGE plpgsql
    SECURITY DEFINER
    SET search_path = public, pg_temp
    AS $$
    DECLARE
      v_uid uuid := auth.uid();
      v_name text := btrim(coalesce(p_name, ''));
      v_org_id uuid;
    BEGIN
      IF v_uid IS NULL THEN
        RAISE EXCEPTION 'not authenticated' USING ERRCODE = '42501';
      END IF;

      IF char_length(v_name) < 1 OR char_length(v_name) > 120 THEN
        RAISE EXCEPTION 'organization name must be 1..120 characters' USING ERRCODE = '22023';
      END IF;

      INSERT INTO public.organizations (name)
      VALUES (v_name)
      RETURNING id INTO v_org_id;

      INSERT INTO public.organization_members (organization_id, user_id, role)
      VALUES (v_org_id, v_uid, 'owner');

      RETURN v_org_id;
    END;
    $$;

    REVOKE ALL ON FUNCTION public.create_organization(text) FROM PUBLIC;
    GRANT EXECUTE ON FUNCTION public.create_organization(text) TO authenticated;

    COMMENT ON FUNCTION public.create_organization(text) IS
      'Authenticated bootstrap: creates an organization and an owner membership for auth.uid() atomically.';
    ```

    If `organizations.name` column is not exactly `name` (e.g. could be `display_name`), or `organization_members` columns differ, adjust the column names to match `20260502000003_organizations.sql` exactly. Do NOT change the function name or signature.
