# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Auth, Org & Dashboard.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind CSS utility classes only) |
| Preset | not applicable |
| Component library | @supabase/auth-ui-react (auth forms only, locked — D-01) |
| Icon library | none for Phase 1 (no icon-dependent UI; avatar uses initials) |
| Font | system-ui, 'Segoe UI', Roboto, sans-serif (system stack — no web font load) |

**Note:** No shadcn. Auth forms are delegated to `@supabase/auth-ui-react` with `ThemeSupa` theme (D-01). All other Phase 1 UI is hand-composed with Tailwind. shadcn may be introduced in Phase 2 for the estimate editor — out of scope here.

**ThemeSupa override tokens:** The ThemeSupa theme will be given a `variables` prop to align its accent color with the product accent (`#2563EB`) and its border radius to `6px`. No other overrides — let ThemeSupa handle form internals.

---

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, badge horizontal padding |
| sm | 8px | Input label-to-field gap, table cell vertical padding |
| md | 16px | Default element spacing, form field gap, card padding |
| lg | 24px | Section padding, nav height internal padding |
| xl | 32px | Above-table area padding, onboarding card vertical padding |
| 2xl | 48px | Auth page vertical centering offset |
| 3xl | 64px | Page-level top margin on auth/onboarding pages |

Exceptions:
- Nav bar height: 56px (not a standard scale step — required for 44px touch target clearance plus 6px vertical padding each side)
- User avatar/initials target: 36px diameter (compact, not a primary touch target)

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Table cell content, form helper text, badge labels |
| Label | 14px | 600 (semibold) | 1.4 | Form field labels, table column headers, nav links |
| Heading | 20px | 600 (semibold) | 1.3 | Page title ("Dashboard", "Create your organization") |
| Display | 28px | 600 (semibold) | 1.2 | Auth page product name "EstimateFlow" only |

Font stack: `system-ui, 'Segoe UI', Roboto, Helvetica Neue, Arial, sans-serif`

**Rules:**
- Body and Label share size (14px) but differ in weight — this is intentional for density.
- Display (28px/600) is used exactly once: the EstimateFlow logo text on the `/auth` page. The larger size (28px vs 20px) provides sufficient visual hierarchy without requiring a third weight.
- No italic. No font sizes outside this table.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` / `slate-50` (`#F8FAFC`) | Page background, table row background, auth page background |
| Secondary (30%) | `#F1F5F9` (slate-100) | Nav bar background, table header row, onboarding card background |
| Accent (10%) | `#2563EB` (blue-600) | See reserved list below |
| Destructive | `#DC2626` (red-600) | Destructive actions only (none in Phase 1 — reserved for Phase 2+) |

**Accent reserved for:**
1. "New Estimate" primary button (background)
2. "Save" / submit button on `/onboarding` form
3. Active nav link underline indicator
4. ThemeSupa override: `default.colors.brand` → `#2563EB`
5. Focus ring on interactive elements (`ring-blue-600`)

**Never use accent for:** body text, table rows, badges, backgrounds, borders, icons in their default state.

**Status badge colors (not accent):**
- Draft: `bg-slate-100 text-slate-600` — neutral gray pill
- Sent: `bg-blue-100 text-blue-700` — muted blue pill (distinct from accent blue; uses blue-100/700 not blue-600)

**Text colors:**
- Primary text: `#0F172A` (slate-900) — headings, table data
- Secondary text: `#64748B` (slate-500) — column headers, helper text, empty state body
- Disabled / placeholder: `#94A3B8` (slate-400)

**Border:**
- Default border: `#E2E8F0` (slate-200) — table dividers, input borders, card borders

## Screen-by-Screen Contract

### `/auth` — Authentication Page

**Layout:** Vertically and horizontally centered card on a `slate-50` background.

**Card:** white background, `rounded-lg`, `shadow-sm`, `border border-slate-200`, max-width 400px, padding `xl` (32px) all sides.

**Above the form:**
- Product name: "EstimateFlow" — Display size (28px/600), `slate-900`, centered, margin-bottom `md` (16px)
- Tagline: "Professional estimates in minutes." — Body size (14px/400), `slate-500`, centered, margin-bottom `xl` (32px)

**Form:** Rendered by `@supabase/auth-ui-react` with `ThemeSupa`. Override variables:
```
variables: {
  default: {
    colors: {
      brand: '#2563EB',
      brandAccent: '#1D4ED8',
    },
    radii: {
      borderRadiusButton: '6px',
      inputBorderRadius: '6px',
    }
  }
}
```

**ThemeSupa handles:** email field, password field, "Sign in" / "Sign up" button, "Forgot password?" link, toggle between sign-in and sign-up modes (D-02).

**Below the card:** No marketing copy, no footer links in Phase 1.

---

### `/auth/callback` — Verification Handler

**Layout:** Full-page centered loading state only. No persistent UI — this route immediately redirects.

**Loading state:** Single line, centered: "Verifying your email..." — Body (14px/400), `slate-500`.

**No error UI on this page** — if the token is invalid, Supabase Auth redirects back to `/auth` with an error query param that ThemeSupa renders inline.

---

### `/onboarding` — Organization Setup

**Layout:** Vertically and horizontally centered card on `slate-50` background. Same card dimensions as auth (max-width 400px).

**Card content (top to bottom):**
1. Heading: "Set up your workspace" — Heading (20px/600), `slate-900`, margin-bottom `sm` (8px)
2. Body: "Your company name will appear on all estimates you send." — Body (14px/400), `slate-500`, margin-bottom `xl` (32px)
3. Label: "Company name" — Label (14px/600), `slate-700`, margin-bottom `xs` (4px)
4. Input: type text, placeholder "e.g. Apex Roofing Co.", full width, `border border-slate-200 rounded-md px-3 py-2`, focus `ring-2 ring-blue-600 border-transparent`
5. Validation error (inline, below input): "Company name is required." — 14px/400, `text-red-600`, margin-top `xs` (4px). Shown only after first submit attempt.
6. Submit button: Label "Create workspace" — full width, `bg-blue-600 hover:bg-blue-700 text-white font-semibold rounded-md py-2`, margin-top `md` (16px)
7. Loading state on submit: button label changes to "Creating..." with cursor-wait; input disabled.

**Cannot be skipped** (D-05): No "skip" link, no secondary action, no back button.

---

### `/dashboard` — Main Dashboard

**Layout:** Full-height page. Top nav bar + main content area below.

#### Top Nav Bar

Height: 56px. Background: `slate-100`. Border-bottom: `1px solid slate-200`.

Left: "EstimateFlow" wordmark — Label size (14px/600), `slate-900`. Padding-left `lg` (24px).

Center: Nav links — "Estimates" | "Settings". Each link: Label (14px/600), `slate-500` default, `slate-900` when active, with a 2px `blue-600` underline on active. Gap between links: `xl` (32px).

Right: User avatar — 36px circle, `bg-slate-200`, user initials in Label (14px/600) `slate-600`. On click: no dropdown in Phase 1 — clicking avatar shows a simple popover with one item: "Sign out" (14px/400, `slate-700`). Padding-right `lg` (24px).

**Org name display:** Not in nav. Shown inline as secondary text below the page heading on the dashboard content area.

#### Dashboard Content Area

Padding: `lg` (24px) horizontal, `xl` (32px) top.

**Page heading row (flex, space-between):**
- Left: Heading "Estimates" — Heading (20px/600), `slate-900`
- Right: "New Estimate" button — `bg-blue-600 hover:bg-blue-700 text-white font-semibold text-sm rounded-md px-4 py-2`

**Org context line (below heading, margin-top `xs` 4px):**
- "{Company Name}" — Body (14px/400), `slate-500`

**Estimates Table:**

Margin-top: `lg` (24px). Full width. `border border-slate-200 rounded-lg overflow-hidden`.

Table header row: `bg-slate-100`. Each `<th>`: Label (14px/600), `slate-600`, text-left, padding `sm` (8px) vertical `md` (16px) horizontal.

Columns (in order, D-08):
1. `Estimate #` — width ~100px
2. `Client` — flexible
3. `Title` — flexible
4. `Status` — width ~100px
5. `Total` — width ~100px, text-right
6. `Last Updated` — width ~140px

Table data rows: white background, `border-t border-slate-200`. Each `<td>`: Body (14px/400), `slate-900`, padding `sm` (8px) vertical `md` (16px) horizontal. Hover: `bg-slate-50`. Cursor: pointer (entire row navigates to estimate editor — stub route for Phase 1).

**Status badge rendering:**
- Draft: `<span class="inline-flex items-center px-2 py-0.5 rounded-full text-xs font-semibold bg-slate-100 text-slate-600">Draft</span>`
- Sent: `<span class="inline-flex items-center px-2 py-0.5 rounded-full text-xs font-semibold bg-blue-100 text-blue-700">Sent</span>`

**Total column:** Display formatted as "$X,XXX" using `formatCents()` from `src/utils/money.ts`. Right-aligned.

**Last Updated column:** Relative time (e.g. "2 days ago") for entries within 30 days; absolute date "Jan 15, 2026" for older. Body (14px/400), `slate-500`.

**Table sorting:** Column headers for `Estimate #`, `Status`, `Total`, and `Last Updated` are clickable to sort. Sorted column header shows `slate-900` text + a small up/down indicator (Unicode ↑ ↓ or ↕, same 14px/600 size, `slate-400` unsorted). No library needed — client-side sort on the fetched array.

#### Empty State (DASH-02)

Shown when the estimates array is empty (first load after onboarding).

Replaces the table. Centered within the content area, margin-top `2xl` (48px).

```
[Icon area: simple dashed border box, 64px × 64px, border-2 border-dashed border-slate-200 rounded-lg — no SVG icon in Phase 1]
```

Below the box (margin-top `md` 16px):
- Heading: "No estimates yet" — Heading (20px/600), `slate-900`, centered
- Body: "Create your first estimate to get started." — Body (14px/400), `slate-500`, centered, margin-top `xs` (4px)
- CTA button: "New Estimate" — same style as header button, centered, margin-top `md` (16px)

#### Loading State (table)

While the estimates fetch is in flight: show 5 skeleton rows. Each skeleton row: full-width `div`, height 40px, `bg-slate-100 animate-pulse rounded`, margin-bottom `sm` (8px). No spinner.

#### Error State (table fetch fails)

Replace table with centered error message, margin-top `2xl` (48px):
- Heading: "Couldn't load your estimates" — Heading (20px/600), `slate-900`, centered
- Body: "Check your connection and try again." — Body (14px/400), `slate-500`, centered, margin-top `xs` (4px)
- Retry button: "Reload estimates" — secondary style: `border border-slate-200 bg-white hover:bg-slate-50 text-slate-700 font-semibold text-sm rounded-md px-4 py-2`, centered, margin-top `md` (16px)
