## Spacing Scale

Declared values (multiples of 4 only, matching Tailwind default scale):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | gap-1 / p-1 | Icon gaps, badge padding, drag handle gap |
| sm | 8px | gap-2 / p-2 | Inline cell padding, action menu item padding |
| md | 16px | gap-4 / p-4 | Default row padding, editor header padding |
| lg | 24px | gap-6 / p-6 | Section header padding, card-level spacing |
| xl | 32px | gap-8 / p-8 | Between sections |
| 2xl | 48px | gap-12 / p-12 | Page-level vertical rhythm (below TopNav) |
| 3xl | 64px | gap-16 | Bottom of page above sticky totals bar |

Exceptions:
- Touch targets for ⋮ action menu button and drag handle (≡): minimum 44px height (Tailwind: `min-h-[44px]`), even if visual element is smaller. Required for mobile usability (web-first, not native, but responsive).
- Sticky totals bar height: 56px fixed (`h-14`) — matches TopNav height for visual balance.
- Offline amber banner height: 40px (`h-10`) — slim, dismissible, not blocking.

## Typography

| Role | Size | Tailwind | Weight | Tailwind | Line Height | Usage |
|------|------|----------|--------|----------|-------------|-------|
| Body | 14px | text-sm | 400 regular | font-normal | 1.5 | Line item description, dropdown options, modal body |
| Label | 12px | text-xs | 600 semibold | font-semibold | 1.4 | Column headers, status badges, save indicator text, "Optional" badge |
| Heading | 16px | text-base | 600 semibold | font-semibold | 1.2 | Section labels, estimate title inline input |
| Display | 20px | text-xl | 600 semibold | font-semibold | 1.2 | Totals bar grand total amount |

Typography is intentionally compact. This is a spreadsheet-like tool, not a reading surface.

Weight rule: Use only 400 (regular) and 600 (semibold). No 500, no 700. Enforced.

Source: Established in TopNav.tsx (`text-sm font-semibold`, `text-sm`).

## Color

| Role | Tailwind Value | Hex Approx | Usage |
|------|---------------|------------|-------|
| Dominant (60%) | bg-white / bg-slate-50 | #ffffff / #f8fafc | Page background, line item row backgrounds |
| Secondary (30%) | bg-slate-100 / border-slate-200 | #f1f5f9 / #e2e8f0 | TopNav, section headers, editor header bar, sticky totals bar, table column headers |
| Accent (10%) | blue-600 | #2563eb | See "Accent reserved for" list below |
| Destructive | red-600 | #dc2626 | Delete line item (in ⋮ menu), delete section confirmation |
| Amber (status only) | amber-500 bg-amber-50 | #f59e0b / #fffbeb | Offline banner background and text only |

Accent (blue-600) reserved for:
- "Send →" primary button (bg-blue-600 text-white)
- "Mark as Sent" confirmation button in modal (bg-blue-600 text-white)
- Active nav link underline border (border-b-2 border-blue-600) — inherited from TopNav
- Focus rings on interactive elements (ring-blue-600) — inherited from TopNav pattern
- "Saved ✓" save indicator text when in success state (text-blue-600)
- "Add line item" button text (text-blue-600, no background)
- "Add section" button text (text-blue-600, no background)

Accent is NOT used for: row hover states, section backgrounds, general borders, labels, or any passive UI element.

Source: StatusBadge.tsx (`bg-blue-100 text-blue-700` for sent), TopNav.tsx (`border-blue-600`, `focus:ring-blue-600`).

All components to build for this phase:

### Editor Layout
- `EstimateEditorPage` — page-level component, full-width single column
- `EditorHeaderBar` — sticky top bar below TopNav: client dropdown + title + save indicator + Send button
- `OfflineBanner` — amber slim banner, dismissible, renders between header bar and sections
- `StickyTotalsBar` — fixed bottom bar: Subtotal / Tax / Total

### Section Components
- `EstimateSection` — wrapper with drag handle (≡), section label (editable), add-line-item button, delete section action
- `SectionHeader` — label input + drag handle + section actions (⋮ menu: rename, delete)
- `LineItemRow` — drag handle + editable cells (Description, Qty, Unit Price, Markup%) + computed total + ⋮ action menu + optional badge
- `LineItemCell` — individual editable cell; activates on click, tabs to next sibling
- `LineItemActions` — ⋮ popover menu: "Mark optional" / "Remove optional" + "Delete"
- `AddLineItemButton` — text button below last line item in section: "+ Add line item"
- `AddSectionButton` — text button below last section: "+ Add section"

### Inline Client Form
- `ClientDropdown` — searchable dropdown for existing clients + "New client" expansion option
- `NewClientInlineForm` — expands within ClientDropdown: Name (required), Email (optional), Phone (optional) + Save / Cancel

### Modals
- `MarkAsSentModal` — confirmation modal: title, body copy, Cancel (ghost) + "Mark as Sent" (primary blue)
- `DeleteSectionModal` — confirmation modal, only shown when section has line items

### Status / Feedback
- `SaveIndicator` — text-only status in editor header (4 states: see Copywriting)
- `OptionalBadge` — `xs` pill: "Optional" label, `bg-slate-100 text-slate-500`
- `DragHandle` — ≡ icon, `text-slate-300 hover:text-slate-500`, always visible (D-11)

## Interaction Contracts

### Line Item Editing (D-09)
- Click any cell in a row: activates edit mode for that entire row. All cells in the row show input borders.
- Focused cell gets `ring-1 ring-blue-600` focus ring.
- Tab order within row: Description → Qty → Unit Price → Markup % → (wraps to Description of next row, or Add line item button if last row).
- Shift+Tab: reverse.
- Enter: commits edit and moves focus to Description of next row.
- Escape: cancels edit, restores previous value, exits edit mode.
- Clicking outside active row: commits edit and exits edit mode.
- Row in edit mode: background `bg-white` with `ring-1 ring-slate-200` outline on the row container.
- Row at rest (not editing): background alternates `bg-white` / `bg-slate-50` (odd/even rows within section).

### Drag and Drop (D-11, EST-10, EST-11)
- Drag handles (≡) always visible on left of every section header and every line item row (not hover-only).
- Handle color at rest: `text-slate-300`. On hover: `text-slate-500`. On active drag: `text-slate-700`.
- Drag ghost: semi-transparent clone of the row (`opacity-50`), no shadow expansion.
- Drop indicator: 2px `bg-blue-600` horizontal line between rows at the drop target position.
- Section drag: entire section (header + line items) moves as a unit.
- Line item drag: constrained to within its parent section. Cannot drag a line item from one section to another (Phase 2 scope).
- `@dnd-kit/core` with `@dnd-kit/sortable` — use `SortableContext` per section for line items, one `SortableContext` for sections.

### Save Indicator (D-12)
Four states, rendered as text in editor header bar, right of title:
- Saving: `text-slate-400 text-xs` — "Saving..."
- Saved: `text-blue-600 text-xs font-semibold` — "Saved ✓"
- Queued: `text-amber-600 text-xs` — "Queued"
- Error: `text-red-600 text-xs` + inline "Retry" link (`text-red-600 underline cursor-pointer`) — "Failed to save · Retry"
- After "Mark as Sent": save indicator is replaced by `StatusBadge` with status='sent'. No further editing; all inputs get `disabled` attribute and `cursor-not-allowed` styling.

### Offline Banner (D-13)
- Appears below EditorHeaderBar when `navigator.onLine === false`.
- `bg-amber-50 border-b border-amber-200 text-amber-800 text-sm` — full width, 40px height (`h-10`), flex-row.
- Left side: "You're offline — changes are saved locally and will sync when you reconnect."
- Right side: dismiss button ("×"), `text-amber-600 hover:text-amber-800`.
- Dismissible per session (not persisted). Returns if user navigates away and back while still offline.
- Does not block editing. No modal, no overlay.

### Mark as Sent Modal (D-14)
- Triggered by "Send →" button in editor header.
- Title: "Mark EST-{number} as sent?"
- Body: "You won't be able to edit it after this."
- Buttons: Cancel (ghost, `border border-slate-200 text-slate-700`) | "Mark as Sent" (primary, `bg-blue-600 text-white`)
- On confirm: status transitions to 'sent', all editor fields disabled, save indicator replaced by StatusBadge.
- Modal overlay: `bg-slate-900/50`, centered card `max-w-sm`, `rounded-lg shadow-lg bg-white p-6`.

### Delete Section (EST-13)
- If section has 0 line items: delete immediately, no modal.
- If section has 1+ line items: show `DeleteSectionModal`.
- Modal title: "Delete section?"
- Modal body: "This will delete {N} line item{s} and cannot be undone."
- Buttons: Cancel | "Delete section" (destructive, `bg-red-600 text-white`).

### Client Dropdown + Inline New Client (D-03)
- Searchable combobox: type to filter existing clients by name.
- At bottom of dropdown list: "+ New client" option (text-blue-600).
- Selecting "+ New client": dropdown closes, inline form expands in place within the header bar area.
- Inline form fields: Name (required, `placeholder="e.g. Apex Roofing Co."`), Email (optional, `placeholder="client@email.com"`), Phone (optional, `placeholder="(555) 000-0000"`).
- Save button: `bg-blue-600 text-white text-sm` — "Save client".
- Cancel: `text-slate-500 text-sm underline` — "Cancel".
- On save: client is created, dropdown shows client name, inline form collapses.

### Photo Attachments (EST-15)
- Photo attachment UI: thumbnail strip below each line item row when attachments exist.
- Thumbnails: 48px × 48px (`w-12 h-12`) rounded-sm, `object-cover`, inline horizontal scroll if more than 4.
- "Attach photo" trigger: small `text-slate-400 text-xs` button at far right of line item row: "Attach photo".
- Upload input: hidden `<input type="file" accept="image/*">`, triggered by click.
- Upload in progress: thumbnail placeholder with `bg-slate-100 animate-pulse`.
- No dedicated attachments section per section — thumbnail strip per line item row is the chosen approach.

- Delete line item: no confirmation modal. Immediate deletion via ⋮ menu "Delete" item. Reversible within session via undo is out of scope for Phase 2 — accept immediate delete.

- Mark as Sent: modal confirmation required (see Interaction Contracts above).

## Layout Spec

### Page Structure (top to bottom)
```
[ TopNav — h-14, bg-slate-100, border-b border-slate-200 ]  ← reuse as-is
[ EditorHeaderBar — h-14, bg-white, border-b border-slate-200, px-6 ]
[ OfflineBanner — h-10, bg-amber-50, border-b border-amber-200 ] (conditional)
[ ReadOnlyBanner — h-10, bg-slate-100, border-b border-slate-200 ] (conditional, after sent)
[ Editor Body — flex-1, max-w-3xl, mx-auto, px-6, py-8, pb-24 ]
  [ EstimateSection × N ]
  [ AddSectionButton ]
[ StickyTotalsBar — fixed bottom-0, w-full, h-14, bg-white, border-t border-slate-200, px-6 ]
```

The editor body uses `pb-24` to ensure the last section is not obscured by the sticky totals bar.

### EditorHeaderBar (D-06)
```
[ ClientDropdown (left, w-48) ]  [ EstimateTitle (center-left, flex-1, mx-4) ]  [ SaveIndicator (right of title, mr-4) ]  [ SendButton (far right) ]
```
- Full bar: `flex items-center h-14 px-6 bg-white border-b border-slate-200`
- ClientDropdown: `w-48` combobox
- EstimateTitle: `flex-1 text-base font-semibold text-slate-900 border-0 bg-transparent focus:outline-none focus:ring-0 placeholder:text-slate-400`
- SaveIndicator: `text-xs mr-4` (state-dependent color)
- SendButton: `bg-blue-600 text-white text-sm font-semibold px-4 py-2 rounded-md hover:bg-blue-700 focus:ring-2 focus:ring-blue-600 focus:ring-offset-2`

### StickyTotalsBar (D-08)
```
[ spacer flex-1 ]  [ Subtotal: $X,XXX.XX ]  [ separator · ]  [ Tax: $0.00] (conditional)  [ Total: $X,XXX.XX (text-xl font-semibold) ]  [ padding right px-6 ]
```
- `fixed bottom-0 w-full h-14 bg-white border-t border-slate-200 flex items-center justify-end gap-6 px-6`
- Label + value pairs: `text-sm text-slate-500` label + `text-sm font-semibold text-slate-900` value
- Grand Total: `text-xl font-semibold text-slate-900`
- Tax row: only shown if any line item has applicable tax (Phase 2: show $0.00 as placeholder if tax column exists in schema; omit row entirely if schema has no tax field)

### EstimateSection
- Section header: `flex items-center gap-3 py-3 border-b border-slate-200`
  - DragHandle: `text-slate-300 hover:text-slate-500 cursor-grab active:cursor-grabbing` — ≡ icon
  - Section label input: `flex-1 text-base font-semibold text-slate-900 border-0 bg-transparent focus:outline-none focus:border-b-2 focus:border-blue-600 placeholder:text-slate-400`
  - Section actions ⋮: `text-slate-400 hover:text-slate-600 ml-auto`
- Section body: line item rows stacked, no padding beyond row padding
- AddLineItemButton: `text-blue-600 text-sm pl-9 py-2 hover:text-blue-700` (indented to align with line item content, past drag handle)

### LineItemRow
- Container: `flex items-center gap-2 px-3 py-2 group` (group enables hover states on child elements)
- At rest: `hover:bg-slate-50`
- In edit mode: `bg-white ring-1 ring-slate-200 rounded`
- Column widths (flex-based, no hard px on Description to allow flex-grow):
  - DragHandle: `w-5 shrink-0 text-slate-300 group-hover:text-slate-500`
  - Description: `flex-1 min-w-0` (grows to fill)
  - Qty: `w-16 shrink-0 text-right`
  - Unit Price: `w-24 shrink-0 text-right`
  - Markup %: `w-20 shrink-0 text-right`
  - Line Total: `w-24 shrink-0 text-right font-semibold text-slate-900` (computed, read-only display)
  - Actions ⋮: `w-8 shrink-0`
- Cell inputs (edit mode): `w-full text-sm bg-transparent border-0 focus:outline-none focus:ring-1 focus:ring-blue-600 rounded px-1`
- Money inputs (Qty, Unit Price, Markup %): `text-right`
- Column header row above first section's line items: `text-xs font-semibold text-slate-400 uppercase tracking-wide` — Description | Qty | Unit Price | Markup | Total | (blank for actions)
